The "Why": Streamlining PM Workflows

Product Management involves diverse tasks, some of which are time-consuming yet crucial, like documentation and issue tracking. Piper Morgan was conceived to explore how LLMs could alleviate some of this burden, freeing up PMs for more strategic activities. The chart below illustrates a hypothetical breakdown of PM time, highlighting areas Piper aims to impact.

By assisting with tasks like drafting GitHub tickets and contextual knowledge retrieval, Piper Morgan aims to reduce time spent on documentation and routine administrative work.

The Journey: From Idea to Prototype

Piper Morgan's development was an iterative process, evolving through distinct sessions rather than formal sprints. Each phase built upon the last, adding new capabilities and refining the assistant's core functionalities.

Session 1: Research & PoC

Evaluated LLM platforms (OpenAI), set up initial environment, and built a basic RAG proof-of-concept for Q&A on synthetic documents.

Session 2: Claude & Enhanced Reasoning

Transitioned to Claude Opus API for improved reasoning quality on complex PM queries. Focused on a vendor-agnostic architecture.

Session 3: GitHub Integration

Developed natural language to GitHub issue drafting, including preliminary logic for labeling and context inference.

Session 4: Web UI & Knowledge Ingestion

Created a Streamlit web interface for easier interaction and integrated ingestion of various organizational document types (PDF, DOCX, MD).

Session 5: Issue Review & Current State

Implemented GitHub issue review suggestions, expanding utility to existing tickets. Culminated in the current prototype capabilities.
